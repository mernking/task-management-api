Task Management API

A RESTful API for managing tasks with user authentication and CRUD operations.

this is developed by DAVID CHI

Installation

Clone the repository:

git clone https://github.com/mernking/task-management-api.git

Navigate to the project directory:

cd task-management-api

Install dependencies:

npm install

Start the server:

npm start

server url:

here is the server url for testing https://task-management-api-2ti2.onrender.com/

API Endpoints

User Endpoints

1. POST /api/auth/signup

Description: Create a new user account.

{
  "username": "exampleUser",
  "password": "examplePassword"
}

2. POST /api/auth/login

Description: Authenticate a user and return a JWT token. 

{
  "username": "exampleUser",
  "password": "examplePassword"
}

Task Endpoints

3. POST /api/tasks

Description: Create a new task.

{
  "title": "Task title",
  "description": "Task description",
  "completed": false,
  "category": "Task category"
}

4. GET /api/tasks

Description: Retrieve all tasks for the authenticated user. 

[
  {
    "title": "Task title",
    "description": "Task description",
    "completed": false,
    "category": "Task category",
    "user": "user_id"
  }
]

5. GET /api/tasks/:id

Description: Retrieve a specific task by ID.

{
  "_id": "task_id",
  "title": "Task title",
  "description": "Task description",
  "completed": false,
  "category": "Task category",
  "user": "user_id"
}

6. PUT /api/tasks/:id

Description: Update a specific task by ID.

{
  "title": "Updated task title",
  "description": "Updated description",
  "completed": true,
  "category": "Updated category"
}

7. DELETE /api/tasks/:id

Description: Delete a specific task by ID.

{
  "message": "Task deleted"
}

Authentication

All task endpoints require authentication. Include the JWT token in the Authorization header of your requests: 

Authorization: Bearer your_jwt_token
